# kcov/ — Generic KCOV Coverage Collection

Lower layer of Trinity's coverage-guided fuzzing: per-child KCOV fd lifecycle, PC-edge hashing into a shared bitmap, KCOV_ENABLE/DISABLE ioctl management (including remote/childop brackets), plateau detection, EBADF forensics, and warm-start bitmap persistence across runs. This is the substrate `cmp_hints/` (comparison-operand hint mining, documented separately in `cmp_hints/CLAUDE.md`) sits on top of.

## Boundary with cmp_hints/

kcov/ owns the KCOV kernel interface end-to-end: opening `/sys/kernel/debug/kcov`, both fd modes (`KCOV_TRACE_PC` and `KCOV_TRACE_CMP`), mmap'd trace buffers, ioctl enable/disable, and the raw PC-edge bitmap (`kcov_shm->bucket_seen[]`) with its own persistence format. `cmp_hints/` never touches an fd or ioctl — it only ever receives an already-mmap'd `unsigned long *cmp_trace_buf` pointer plus `(nr, do32)` from kcov/ and mines it for kernel comparison constants.

The call chain per syscall: `random_syscall/dispatch.c` calls `kcov_collect()` (PC path) and, for CMP-mode children, `kcov_collect_cmp()` — kcov/collect.c:776. `kcov_collect_cmp()` validates the cmp trace buffer, then calls straight into `cmp_hints_collect()` (cmp_hints/collect.c), which does the actual per-record parsing, constant filtering, and pool insertion. kcov/ never parses a CMP record itself except in the childop-local fast path in enable.c's `childop_cmp_collect()`, which duplicates a narrow slice of the record-decode logic (type/size/arg1/ip extraction, boring-constant filter) so childop syscalls can insert into `cmp_hints_childop_insert()` without depending on cmp_hints.c's hot-loop internals.

Two pieces of shared machinery cross the boundary explicitly:
- **KASLR canonicalisation**: `kcov_kaslr_base` is resolved once in kcov/lifecycle.c and exposed both as a direct extern (used by kcov/collect.c's `kcov_canon_pc`) and via the accessor `kcov_kaslr_base_value()` (used by cmp_hints/persist.c to stamp/validate its own on-disk header). `kcov_canon_cmp_ip()` (kcov/collect.c:75) is the single canonicalisation entry point for CMP-instruction addresses; cmp_hints/collect.c calls it before every pool insert.
- **Mode/lifecycle coupling**: `struct kcov_child` (mode, fd, cmp_capable, bracket_owned) lives in kcov's domain (`include/kcov.h`/`kcov-internal.h`) but is read by cmp_hints indirectly through the `is_explorer`/`strategy_at_pick` args passed into `kcov_collect_cmp()` from `random_syscall/dispatch.c`.

Persistence is entirely separate: kcov/persist.c persists the PC-edge bitmap (`kcov-bitmap-*` cache file); cmp_hints/persist.c persists the CMP constant pools (separate file), independently versioned, sharing only the KASLR-base agreement check pattern.

## Key design decisions

1. **KASLR-invariant edge hashing** — every PC is canonicalised (`pc - kcov_kaslr_base`) before hashing, so the bucket-seen warm-start cache and cmp-hint pools stay valid across kernel reboots of the same build. `kcov_kaslr_base` is resolved pre-fork in the parent and inherited via COW.
2. **AFL-style hit-count bucketing with dedup** — `kcov_collect()` maintains a per-child open-addressed dedup table (generation-stamped instead of memset per call) so a repeated hit on the same edge advances through 8 count buckets (1,2,3,4-7,8-15,16-31,32-127,128+) rather than just registering "seen".
3. **Shadow transition coverage** — a second hash of `(prev_canon_pc, cur_canon_pc)` pairs into a separate bitmap (`transition_seen[]`), gated by `kcov_transition_coverage_mode` (default SHADOW: observed but not fed into steering) and `kcov_transition_reward_mode` (default COMBINED: blended into frontier/bandit scoring). Both are rollback-able via runtime knobs; the design doc comment in kcov.c documents the A/B evidence that justified promoting COMBINED to the live default.
4. **Two-fd-per-child model** — each child opens both a PC fd and a dedicated CMP fd at init, probes CMP support, then commits to exactly one mode (`KCOV_MODE_PC` or `KCOV_MODE_CMP`) for its lifetime (`kcov_init_child_select_mode`, ratio-gated by `KCOV_CMP_CHILD_RECIPROCAL`). The kernel enforces one enabled fd per task (`-EBUSY` on double-enable), so this is a hard constraint, not just an optimisation.
5. **Bounded EBADF recovery, not silent masking** — `kcov_enable_trace`/`kcov_enable_cmp`/`kcov_enable_remote` retry a vanished fd via `kcov_recover_fd()` up to `KCOV_RECOVERY_MAX` times (counter resets only on an actual coverage harvest, not just a successful ioctl, closing a "recover then immediately re-EBADF" budget-drain loop); exhaustion `_exit()`s with a distinct code so the parent reaper's fast-die circuit breaker engages instead of the child looping degraded forever.
6. **fd relocation to keep kcov out of the fuzzer's own fd space** — both kcov fds are `F_DUPFD_CLOEXEC`'d up to 60000 after setup, because the low fd numbers the kernel initially hands back collide with the working set trinity's own generators pick from (live-fd rings, typed-fd rerolls).
7. **Graduated cold-syscall skip** — `kcov_syscall_cold_skip_pct()` has a hard saturation cap (zero edges after N calls, or a call:edge ratio past a threshold → flat high skip %) plus a graduated ramp (50% baseline rising to 90% cap) based on calls-since-last-new-edge, so dead syscalls are throttled but never fully starved (kernel state can change underneath a long run).
8. **7-version warm-start format with per-version compatibility windows** — persist.c accepts versions 5–7 on load; each version bump documents exactly which prior versions still load and what silently degrades (e.g., v5 loads on a v7 binary but lacks the diag/strat trailer blocks). The save path always recounts `bucket_seen[]` from scratch rather than trusting running atomics, so the on-disk (counters, bitmap) pair is CRC-coherent by construction.
9. **Persist-refusal instead of silent corruption round-trip** — if a recount at save time falls below the warm-load floor or diverges materially from the live `edges_found` atomic, the save is refused (prior on-disk file kept) rather than persisting a scribbled bitmap; refusal is logged and after 1000 consecutive refusals the run requests a clean shutdown (`EXIT_SHM_CORRUPTION`) since the underlying scribble is not self-healing.
10. **First-EBADF forensic latch is one-shot and CAS-gated** — only the very first EBADF observed across the whole run gets the expensive chronicle walk (backward scan of the fd-mutating syscall ring, `/proc/self/fd` snapshot via raw `getdents64`); subsequent EBADFs just bump counters. Two separate backward walkers (`kcov_find_last_fd_mut_slot` broad, `kcov_find_last_closer_slot` narrow) are captured together so a benign allocator (dup/F_DUPFD) masking the real closer further back in the ring is visible by comparing the two results.
11. **Plateau detection uses `distinct_edges`, not `edges_found`** — `edges_found` increments on every bucket-bit transition (including bucket churn on already-discovered edges) and never truly flattens; `distinct_edges` increments only on first-sighting of an edge, so its per-window delta is the correct "has discovery stalled" signal. Hysteresis band (ENTER threshold < EXIT threshold) prevents flapping.
12. **childop CMP harvest duplicates a narrow decode slice rather than sharing cmp_hints' hot loop** — `childop_cmp_collect()` in enable.c re-implements record type/size/arg1/ip extraction and the "boring constant" filter locally so the childop wrapped-syscall critical section doesn't pull in cmp_hints.c's full machinery; it explicitly does not replicate the wider A/B telemetry filter, since this lane has no A/B yet.

## Integration points

- `random_syscall/dispatch.c` — main per-syscall driver: calls `kcov_collect()` for PC coverage and `kcov_collect_cmp()` for CMP-mode children; reads `new_edge_count`/`struct kcov_pc_result` out-params
- `child-init.c` — calls `kcov_init_child()` once per forked child (after stdio dup2, before the syscall loop)
- `child.c`, `syscall.c` — call the enable/disable/bracket API around syscall dispatch and extra-fork paths (`kcov_note_extrafork`)
- `child-altop.c`, `child-canary.c` — childop callers of the PC/CMP bracket API for non-syscall child operations
- `signals.c` — `child_fault_handler` siglongjmp target for the `CONFIG_GUARD_SHARED` protection-strip fault detector in `kcov_enable_trace`
- `utils/range_overlap.c` — mm-syscall sanitiser (`range_overlaps_shared`) consults the shared-region tracker that kcov/lifecycle.c registers trace buffers into, so fuzzed `munmap`/`mprotect`/`madvise` can't corrupt kcov's own mmaps
- `cmp_hints/collect.c`, `cmp_hints/persist.c`, `cmp_hints/cmp_hints.c` — consumers of `kcov_canon_cmp_ip()` and `kcov_kaslr_base_value()`; see Boundary section above
- `strategy.c`, `strategy/strategy-plateau.c`, `strategy/strategy-frontier.c` — `strategy_plateau_response()` invoked from `kcov_plateau_check`; frontier scoring reads `frontier_record_new_edge()`/transition deltas that kcov/collect.c feeds
- `trinity.c`, `main/main.c`, `main/main-spawn.c` — warm-start `kcov_bitmap_load_file`/`save_file` at boot/shutdown, `kcov_bitmap_enable_snapshots` setup
- `stats/dump.c`, `stats/log.c`, `stats/json/kcov.c` — periodic stats dump splices in `kcov_pc_diag_format`/`kcov_cmp_diag_format` output and per-syscall call/edge counters
- `args/generate-args.c`, `args/cmp_hint_inject.c` — the `cmp_hint_injected_this_call` latch that `kcov_collect()`'s found-new-edge branch reads to credit a PC-edge win back to the cmp-hint pipeline
- `random_syscall/strategy-accounting.c` — reads kcov_shm per-syscall/per-strategy counters for bandit/frontier bookkeeping; also a caller of the bitmap snapshot path

## Areas of attention

1. **persist.c is 1,347 LOC for what is conceptually one file format** — seven on-disk versions, each with its own compatibility branch on both save and load, plus an embedded SHA-256 implementation, plus the corruption-refusal/canary machinery. The version-compatibility logic (v5/v6/v7 conditional trailer reads) is correct but dense; a version bump requires touching the header struct, two `_Static_assert`s, both save and load functions, and the min-load-version constant in lockstep.
2. **collect.c's `kcov_collect()` (lines 237–774) is a single ~540-line function** covering PC-walk, dedup, bucketing, transition hashing, per-syscall/mode/childop counter fan-out, and stats-ring flush triggering. Heavily commented and each step is individually simple, but the function has many early-exit and gated-counter branches that would be easy to desynchronize (e.g. the `attempts == bracketed + sum(skipped)` invariant mentioned in enable.c comments is maintained by convention across multiple functions, not enforced by a single accounting structure).
3. **EBADF recovery and forensics (enable.c, lifecycle.c, diag.c) implement the same close-race defense three times** — PC enable, CMP enable, and remote-fallback-to-PC each have their own retry loop with near-identical structure (EINTR retry cap, diag record, EBADF branch calling `kcov_recover_fd`, exhaustion `_exit`). This is deliberate (each fd pair has slightly different failure semantics — e.g. CMP non-EBADF failure demotes to PC-only rather than dying) but is a maintenance-parallel-code-paths risk if the recovery contract changes.
4. **plateau.c is not itself complex machinery** — despite the name suggesting statistical/heuristic sophistication, `kcov_plateau_check()` is a straightforward hysteresis-banded threshold state machine on a windowed delta (no regression, no smoothing, no adaptive thresholds). The more intricate code in this file is `kcov_covjump_breadcrumb_maybe()`, which is pure diagnostics (no behavior gate) but does nontrivial work per emission: top-N insertion sort over childop deltas, multiple snapshot/delta computations across `minicorpus_shm`/`chain_corpus_shm`/childop-invocation arrays, and string formatting into three fixed buffers.
5. **Corruption-defense pattern is asymmetric with cmp_hints/** — kcov/'s bitmap has a canary check (`kcov_bitmap_canary_check`) and a persist-refusal floor, but there is no per-access `kcov_shm` corruption gate analogous to cmp_hints' `cmp_hints_pool_corrupted()` sentinel checks; `bucket_seen[]` is a flat byte array with no canary bytes flanking it, only a periodic full-scan popcount comparison.

## Summary

kcov/ is the KCOV kernel-interface layer: it owns every ioctl, every mmap, both trace-buffer fds per child, PC-edge hashing into a shared bitmap with AFL-style bucketing plus a shadow transition-pair signal, plateau detection over the resulting edge-discovery curve, and warm-start persistence of the bitmap across runs. It hands cmp_hints/ nothing but a raw mmap'd CMP trace buffer and a KASLR-canonicalisation utility function — the actual mining of comparison constants, hypothesis inference, and re-injection into arg generation is entirely cmp_hints/'s responsibility. The two subsystems are cleanly layered (kcov below, cmp_hints above) but share the KASLR-base value as their one piece of common state.
