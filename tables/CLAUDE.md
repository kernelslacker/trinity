# tables/ — Syscall Table Registry

Central registry that turns the 353 per-syscall descriptors under `syscalls/` (and the per-arch static arrays under `include/syscalls-*.h`) into the live, shared-memory-backed arrays every picker, dispatcher, and stats/hint subsystem indexes by syscall number.

## Files (3 files, ~1,725 LOC)

| File | Lines | Role |
|---|---|---|
| tables.c | 1038 | Arch-agnostic core: table copy into shared memory (`copy_syscall_table()`), EXPENSIVE bitmap build, activate/deactivate with cost-pool bookkeeping, lookup (`get_syscall_entry()`, `search_syscall_table()`), sanity checks, `munge_tables()` orchestration, biarch/uniarch dispatch shims |
| tables-uniarch.c | 229 | Single-table arch variant: `syscalls` global, activate/deactivate/toggle/enable-random/group-setup/dump/display, all operating on one flat table |
| tables-biarch.c | 458 | Dual-table (32-on-64) arch variant: `syscalls_32bit`/`syscalls_64bit` globals, same operation set doubled, plus 32/64 pairing logic in `enable_random_syscalls_biarch()` and `toggle_syscall_biarch()` |

## Data model

- `struct syscalltable { struct syscallentry *entry; }` (`include/syscall.h:684`) — one-pointer wrapper; the per-arch static arrays (`syscalls_x86_64[]`, `syscalls_i386[]`, `syscalls_aarch64[]`, etc. in `include/syscalls-*.h`) are `struct syscalltable[]` literals populated from the descriptors under `syscalls/`. `include/arch-syscalls.h` picks the right header via `#ifdef __x86_64__` / `__aarch64__` / etc.
- `include/arch-<arch>.h` `#define`s `SYSCALLS` (uniarch) or `SYSCALLS32`/`SYSCALLS64` (biarch, only x86-64 today) to name the arch's static array(s).
- `select_syscall_tables()` (tables.c:743) is the one-time init entry point: it calls `copy_syscall_table()` on `SYSCALLS`/`SYSCALLS32`/`SYSCALLS64`, storing the result in the arch-agnostic globals `syscalls` / `syscalls_32bit` / `syscalls_64bit` declared in `include/tables.h`, and sets `max_nr_syscalls` / `max_nr_32bit_syscalls` / `max_nr_64bit_syscalls` from `ARRAY_SIZE()` of the source array. `bool biarch` is the global that every other function in tables.c branches on to pick uniarch vs biarch behavior.
- `copy_syscall_table()` (tables.c:662) allocates a shared (`alloc_shared()`) array of `struct syscallentry` sized to the *non-NULL* entry count, memcpy's each static descriptor into it, and stamps derived fields once: `number`, category, five name-comparison booleans (`is_mmap2`, `is_execve`, etc. — precomputed so hot-path checks in `.sanitise`/`.post` hooks are a byte load instead of a strcmp), and four precomputed arg-scrub bitmasks (`address_scrub_mask`, `cleanup_arg_mask`, `fd_arg_mask`, `len_arg_mask`) plus `nested_address_scrub_mask`. This shared copy — not the static const array — is what children mutate (flags, active_number) at runtime, so state (e.g. `-c`/`-x` toggles, active-list membership) is visible fleet-wide.
- `MAX_NR_SYSCALL` (`include/syscall.h:13`, `#define MAX_NR_SYSCALL 1024`) is a fixed shared-memory sizing constant — NOT the same as `max_nr_syscalls`/`max_nr_*bit_syscalls`, which are the *actual* per-arch table sizes (`ARRAY_SIZE()` of the arch array, typically 300-450). `MAX_NR_SYSCALL` bounds the shm arrays declared in `include/shm.h` (`active_syscalls[MAX_NR_SYSCALL]`, `active_cheap[MAX_NR_SYSCALL]`, `active_expensive[MAX_NR_SYSCALL]`) and the cmp_hints/kcov/minicorpus per-syscall pool dimensions (`cmp_hints.h`, `kcov.h`, `minicorpus.h` all index `[MAX_NR_SYSCALL]`). `activate_syscall_in_table()` (tables.c:190) asserts `nr_active + 1 <= MAX_NR_SYSCALL` and hard-exits if the real table ever grows past the shm cap.
- Per-syscall runtime state lives in three parallel shm views, kept in lock-step: the flat `active_syscalls[]` (all active syscalls, used by pickers for uniform draws) and the `active_cheap[]`/`active_expensive[]` cost pools (partitioned by the `EXPENSIVE` flag, used by the cost-aware picker in `random_syscall/pick-common.c`). `assert_pool_partition()` (tables.c:162) panics if `cheap + expensive != active` — a lock-step invariant enforced on every activate/deactivate.
- `expensive_bits_uniarch`/`_32`/`_64` — read-only bitmaps (`uint64_t[EXPENSIVE_BITMAP_WORDS]`), built once by `build_expensive_bitmap()` in `select_syscall_tables()`, queried by `syscall_is_expensive()` — the single source of truth for cost-pool routing so pickers can reject expensive picks without touching the syscallentry cache line.

## Key design decisions

1. **Static arch table -> shared mutable copy.** The compile-time `SYSCALLS*` arrays are `const`; `copy_syscall_table()` produces the mutable shm copy every child operates on, so flag changes (AVOID, TO_BE_DEACTIVATED, active_number) are visible across the fleet without IPC.
2. **Biarch vs uniarch is a single runtime bool (`biarch`), not a compile split at call sites.** Nearly every public function in tables.c (`init_syscalls`, `toggle_syscall`, `dump_syscall_tables`, `display_enabled_syscalls`, `setup_syscall_group`, `deactivate_disabled_syscalls`) is a two-line `if (biarch) ... else ...` shim delegating to the arch-specific file. Only `ARCH_IS_BIARCH` (a `#ifdef`, set for x86-64) gates which set of globals (`syscalls` vs `syscalls_32bit`/`syscalls_64bit`) actually exists — `biarch` itself is set at runtime in `select_syscall_tables()`.
3. **O(1) activate/deactivate via swap-with-last.** Both the flat `active_syscall[]` and the cheap/expensive pools use `entry->active_number`/`entry->pool_number` (1-based, 0 = inactive) as a back-index, so removing an entry swaps in the last element and rewrites one neighbour's back-index — no O(N) memmove despite the pools living in shared memory read by concurrent children.
4. **Cost-pool bitmap is authoritative, never inferred.** `cost_pool_of()` always consults the precomputed `EXPENSIVE` bitmap, never a pool array's current membership — the comment in tables.c (lines 72-89) explicitly calls out that inferring from live-mutable state would let a torn pool silently "ratify" corruption instead of tripping `assert_pool_partition()`.
5. **`get_syscall_entry(callno, do32)`** (tables.c:998) is the universal accessor abstracting biarch away — compiled to either the direct uniarch path or the do32-branching biarch path via `#ifndef ARCH_IS_BIARCH`. Every consumer outside tables/ goes through this rather than indexing `syscalls[]` directly.
6. **`this_syscallname()`** caches the resolved entry on `this_child()->syscall.entry` (stamped by `dispatch_step()`) instead of re-running `get_syscall_entry()` — called from `.sanitise`/`.post` hooks, some of which (mmap/mmap2) fire twice per call.
7. **`-x` exclusion uses a persistent flag, not a derived state.** `EXPLICITLY_EXCLUDED` is set by `toggle_syscall_n()`/`toggle_syscall_biarch_n()` and deliberately survives `deactivate_disabled_syscalls()` (which only clears `ACTIVE|TO_BE_DEACTIVATED`), so `syscall_nr_is_excluded()` can answer correctly for raw `syscall(__NR_x,...)` call sites in childops regardless of `-c`/`-r`/`-g` targeting — the file's comment (tables.c:929-966) documents a prior bug where inferring exclusion from `!ACTIVE` misreported non-targeted syscalls.
8. **Group inheritance is a flat parent lookup**, not a tree walk: `group_parent[NR_GROUPS]` (tables.c:774) only has one non-default entry (`GROUP_XATTR -> GROUP_VFS`), checked as `entry->group == group || group_parent[entry->group] == group`.

## Integration points

- `random_syscall/pickers.c`, `random_syscall/pick-common.c`, `random_syscall/dispatch.c` — the actual runtime consumers: index `child->active_syscalls[]` (a per-child pointer set once to `shm->active_syscalls`/`32`/`64` in pick-common.c) to pick a syscall number, then call `get_syscall_entry()` and `syscall_is_expensive()` (both defined here) to resolve the descriptor and cost-pool bit
- `args/generate-args.c`, `kcov/diag.c`, `cred_throttle.c`, `main/reap.c`, `pre_crash_ring.c`, `post-mortem.c`, `results.c`, `minicorpus.c`, `sequence.c`, `strategy/strategy-frontier.c`, `utils/debug.c` — all call `get_syscall_entry(nr, do32bit)` to resolve a `struct syscallentry*` from a raw syscall number; strategy-frontier.c notes it deliberately bypasses this indirection on its hot path by caching the entry
- `include/shm.h` — owns the shared arrays this module maintains: `active_syscalls[32|64][MAX_NR_SYSCALL]`, `active_cheap[32|64]`/`active_expensive[32|64]`, their `nr_active*` counters, and `syscalltable_lock` (taken by `deactivate_syscall_locked()` for picker-side concurrent deactivation)
- `include/cmp_hints.h`, `include/kcov.h`, `include/minicorpus.h` — size their own per-syscall pools/rings to `MAX_NR_SYSCALL` (not `max_nr_syscalls`), the shared sizing constant this module's `activate_syscall_in_table()` guards against overflowing
- `syscall-gate.h` / `trinity_raw_syscall()` — consults `syscall_nr_is_excluded()` (tables.c) to turn a raw `syscall(__NR_x,...)` in a childop into an ENOSYS instead of an actual call when `-x` named it
- `include/syscalls-*.h` (one per arch) — the compile-time `struct syscalltable` literal arrays this module copies into shm; selected via `include/arch-syscalls.h`'s `#ifdef __<arch>__` chain and named via each `include/arch-<arch>.h`'s `SYSCALLS`/`SYSCALLS32`/`SYSCALLS64` macro
- `params.c` — `-c`/`-x`/`-r`/`-g` command-line flags drive `toggle_syscall()`, `enable_random_syscalls()`, `setup_syscall_group()`, all dispatched from `munge_tables()` (tables.c:884), called once after arg parsing

## Areas of attention

1. **Three parallel shm state arrays kept in lock-step by convention, not by structure.** `active_syscalls[]`, `active_cheap[]`, `active_expensive[]` (plus their `32`/`64` biarch twins — 9 arrays total when biarch) are separate `int[MAX_NR_SYSCALL]` shm fields updated together in `activate_syscall_in_table()`/`deactivate_syscall_in_table()`; `assert_pool_partition()` only catches drift by exit(EXIT_FAILURE), it can't prevent it. A bug in either function that updates one array but not another is a hard fleet-wide crash, not a silent corruption — by design (`assert_pool_partition()`'s comment explicitly rejects "silently ratify" as an option).
2. **MAX_NR_SYSCALL is a hardcoded ceiling checked against but not derived from the real table.** `include/cmp_hints.h:620` has `_Static_assert(MAX_NR_SYSCALL == 1024, ...)` — the value is load-bearing for shm layout elsewhere in the codebase, so bumping it (or growing an arch table past it) is a coordinated multi-file change, not a local one; tables.c only guards it at runtime (`exit(EXIT_FAILURE)` in `activate_syscall_in_table()`), not at compile time.
3. **biarch/uniarch duplication.** tables-biarch.c essentially doubles every uniarch function (`activate_syscall32`+`activate_syscall64` vs `activate_syscall`, etc.) rather than parameterizing over table pointers; `activate_syscall_in_table()`/`deactivate_syscall_in_table()` in tables.c already take table+counter pointers as parameters specifically to let both variants share one implementation, so the duplication is confined to the thin wrapper layer, not the core logic.
4. **`copy_syscall_table()` is the single point mutating derived per-entry fields at init** (masks, category, name-comparison booleans) — any new precomputed field needs to be added there and only there, and its zero-init assumption (`results_init_one()` stamping min>max sentinels because `alloc_shared()` zero-fills) is easy to miss if a new field also needs a non-zero "unset" sentinel.

## Summary

tables.c holds the arch-agnostic machinery (shm copy-out, EXPENSIVE bitmap, activate/deactivate with cost-pool bookkeeping, `get_syscall_entry()` lookup, `munge_tables()` startup orchestration); tables-uniarch.c and tables-biarch.c are thin per-arch-shape implementations of the same operation set (activate/deactivate/toggle/enable-random/group-setup/dump/display) operating on one table vs a 32-bit/64-bit pair. `MAX_NR_SYSCALL` (1024) is a fixed shm-sizing ceiling shared with cmp_hints/kcov/minicorpus, distinct from the real per-arch table sizes (`max_nr_syscalls` etc.) computed here at init. Every syscall-number-to-descriptor resolution outside this directory goes through `get_syscall_entry()`, making this the sole authority pickers, dispatch, args generation, and stats/hint subsystems rely on for syscall metadata.
