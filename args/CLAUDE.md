# args/ — Syscall Argument Generation Layer

Decides what value goes in each of a syscall's up to 6 argument slots, given the slot's declared `argtype` (ARG_FD, ARG_STRUCT_PTR_IN, ARG_RANGE, ...) and, for struct-shaped slots, the per-field schema pulled from struct_catalog/. Runs once per dispatch from `generate_syscall_args()`, between sanitise-callback selection and the kernel call.

## Key design decisions

1. **Descriptor-table dispatch (argtype_table.c)** — every argtype maps to one `.generate` function pointer plus flags (`cleanup`, `paired_length`, `default_address_scrub`, fd-bias eligibility). `fill_arg()` is the single call site; adding an argtype means adding one table row, not touching a switch statement anywhere else. Masks derived from the table (`address_scrub_mask`, `cleanup_arg_mask`, `fd_arg_mask`, `len_arg_mask`) are precomputed once per `syscallentry` at table-init time (tables/tables.c:707-723) so per-dispatch code does bit-scans, not per-slot `argtype_get_ops()` calls.
2. **Three-pass struct fill (struct_fill.c)** — scalar tags first (order-independent), then pointer tags (allocate sub-buffers, remember chosen size/count in `chosen_len[]` keyed by field index), then length tags (read back `chosen_len[]` for the paired buffer). Resolves PTR/LEN cross-field coupling without a topological sort. A pre-pin sub-pass handles multi-sibling LEN fields (one shared count for several buffer fields, e.g. kprobe_multi's syms+addrs+cookies).
3. **Schema-aware vs. raw splat is byte-identical for unannotated structs** — every catalog field defaults to FT_RAW, so `fill_field_raw()` produces the same rand32()-per-field sequence the pre-schema code did. Individual structs opt into tag-specific mutators (FT_FLAGS, FT_ENUM, FT_RANGE, ...) incrementally.
4. **Post-fill mutation is a separate, gated pass (struct_mutate.c)** — after schema-fill produces a validator-passing struct, `struct_field_mutate_one()` (12% chance) walks a skip-listed candidate set (PTR/LEN/FD/ADDRESS/BPF_PROGRAM/TAGGED_UNION excluded by allow-list, not deny-list) and applies exactly one tag-respecting neighbour mutation (bit flip, enum swap, range ±1 step) so a coverage win attributes to one (tag, field) pair.
5. **Variant resolution mirrored across three independent walkers** — fill (struct_fill.c's `struct_variant_overlay_nested`), mutate-candidate-collection (struct_mutate.c), and scrub (scrub.c's `scrub_variant_overlay_nested`) all resolve tagged-union variants (variant->fields, variant->base->fields, nested_variant->fields) the identical way. A mismatch between any two would silently mis-scope fills, mutations, or the address scrub — struct_mutate.c has an explicit selftest (`selftest_variant_address_walk`) pinning this invariant for the scrub side.
6. **CMP-hint injection is scattered across 6+ call sites, centralized in one gate/credit pair** — `cmp_hint_baseline_should_inject()` / `cmp_hint_inject_denom()` (cmp_hint_inject.c) own the rate math (baseline 1/16, A/B arm 1/12, plateau-amplified 1/4); `credit_cmp_hint_injection()` owns the bookkeeping. Call sites in handle_arg.c (ARG_RANGE/OP/LIST), gen_arg_scalar.c (ARG_UNDEFINED), gen_arg_struct_ptr.c (ARG_STRUCT_SIZE), and gen_arg_time.c (field-scoped timespec) each just call the gate then the credit function.
7. **Two-tier address safety net** — `blanket_address_scrub()` (generate-args.c) covers the top-level rec->a1..a6 slots flagged `default_address_scrub` in the table; `nested_address_scrub()` (scrub.c) recurses inside cataloged structs to find FT_ADDRESS fields the blanket pass can't see. Both run twice per dispatch: once post-sanitise here, once again in syscall.c's `__do_syscall()` after the sanitise callback has had a chance to relocate pointers.
8. **`arg_meta_init()`'s generation-stamped sidecar (generate-args.c) is SHADOW-only** — comments explicitly note no caller of `arg_meta_init` consults the dir/owner classification for a decision yet; it exists to build telemetry (`arg_meta_addr_with_meta` / `_without_meta`, `arg_meta_scrub_would_destroy_in`) ahead of a future consumer.
9. **CSFU bucket model (csfu.c) is 60/10/10/10/10** — EXACT dominates so the syscall reaches post-validator code the majority of the time; the four mutation buckets exercise `copy_struct_from_user()`'s size-word branches (undersize reject, oversize-zero-tail-OK, oversize-nonzero-tail-E2BIG, tail-byte-off-by-one).

## Integration points

- `syscall.c` — `__do_syscall()` calls `validate_arg_coupling()` post-sanitise, then a second `blanket_address_scrub()` pass after the per-syscall sanitise callback has run, immediately before entering the kernel.
- `random_syscall/dispatch.c` — the sole caller of `generate_syscall_args()` (3 call sites: normal dispatch, minicorpus replay retry, re-exec regeneration); also calls `output_syscall_prefix()`/`output_syscall_postfix()` from arg-decoder.c.
- `tables/tables.c` — `copy_syscall_table()` calls the `compute_*_mask()` / `populate_arglist_all_bits()` helpers from argtype_table.c once per syscallentry at table-init time to cache the scrub/cleanup/fd/len bitmaps consumed every dispatch.
- `struct_catalog/` — the schema source of truth: `struct_desc`, `struct_field`, `field_tag`, `union_variant` types and `struct_catalog_lookup()`/`struct_arg_lookup()` are defined there and consumed throughout struct_fill.c, struct_mutate.c, struct_validate.c, scrub.c, gen_arg_struct_ptr.c, gen_arg_time.c, argtype_table.c. `struct_catalog/sockaddr-sockopt.c` and `syscalls/setsockopt.c`, `syscalls/bpf.c` call back into `struct_field_fill_schema_aware()`.
- `cmp_hints/` — args/ is a heavy consumer, not a producer: handle_arg.c, gen_arg_scalar.c, gen_arg_struct_ptr.c, gen_arg_time.c call `cmp_hints_try_get()`/`cmp_hints_try_get_ex()`/`cmp_hints_field_try_get()` (31 call sites total) to pull kernel-observed constants into ARG_RANGE/OP/LIST/UNDEFINED/STRUCT_SIZE/TIMESPEC slots; cmp_hint_inject.c centralizes the injection-rate gate and credit bookkeeping that cmp_hints/collect.c and stats/kcov_cmp.c read back.
- `deferred-free.c` — `zmalloc_tracked()`/`deferred_free_enqueue()`/`deferred_free_enqueue_or_leak()` back every heap allocation this directory hands to the kernel (struct buffers, iovecs, pathnames, sockaddrs, csfu buffers, valresult buffers).
- `prop_ring.c` — handle_arg.c (ARG_OP) and gen_arg_scalar.c (ARG_UNDEFINED, and typed consumers for key-serial/timerid/sem/msg/shm ids) pull recently-observed-return values via `prop_ring_try_get()`/`prop_ring_try_get_kind()`, A/B-gated per child.
- `syscalls/*.c` — per-syscall `.sanitise` callbacks run after `generic_sanitise()`'s `fill_arg()` loop and may overwrite slots; several files (getresuid.c, getresgid.c, timer_create.c, pipe.c, socketpair.c, poll.c, io_setup.c, memfd_create.c, futex.c, socketcall.c) have explicit comments reasoning about ordering against `blanket_address_scrub`. csfu.c is consumed directly by clone3.c, mount.c, statmount.c, listmount.c, listns.c, open_tree_attr.c, perf_event_open.c, sched_setattr.c, and others; valresult.c by getsockopt.c, getpeername.c, getsockname.c, recv.c, lsm_list_modules.c; output-poison.c by newfstat.c.
- `stats/kcov_cmp.c` — reads `blanket_address_scrub_slots_walked` and the cmp-hint injection counters for periodic/JSON dumps.

## Areas of attention

1. **struct_mutate.c is 1210 lines, but ~700 are self-test code** (`selftest_flags` through `selftest_variant_address_walk`, called once at startup from `struct_field_mutate_self_check()` since trinity has no separate unit-test binary). The live logic (candidate collection, weighted pick, per-tag dispatch) is a tighter ~500 lines, but the file reads as much larger than its functional surface.
2. **Three independent variant-resolution walkers must stay in lockstep** — struct_fill.c's `struct_variant_overlay_nested`, struct_mutate.c's `collect_mutable_candidates`, and scrub.c's `scrub_variant_overlay_nested` each re-implement the same "resolve variant, then base, then nested" traversal. A change to variant resolution semantics (struct_catalog's `struct_desc_resolve_variant`/`struct_desc_resolve_nested_variant`) needs verification against all three call sites, not just one; a missed FT_ADDRESS field in the scrub walker is a use-after-relocate / cross-child aliasing risk, not just a coverage gap.
3. **scrub.c's `nested_scrub_base_unsafe()` and `range_readable_user()` gates exist because pointers can be stomped between sanitise and dispatch** — the comments describe a real race window (sibling child scribbling a shared buffer) that the pre-deref shape/range checks narrow but do not eliminate; a missed page-fault race still SEGVs (matching the same risk class documented in cmp_hints/CLAUDE.md's field.c entry).
4. **`fill_arg()`'s central-generator address-family meta stamping (argtype_table.c:565-594) is a SHADOW telemetry-only feature layered onto the hot path** — it runs on every ARG_ADDRESS/ARG_NON_NULL_ADDRESS fill but has no live consumer yet; a future consumer of `arg_meta.dir`/`.owner` needs to audit that the "central-generator coverage" classification here and the `argtype_default_meta()` seed in generate-args.c agree, since both write to the same `arg_meta[]` slot with different generation-tagging rules.
5. **CSFU and struct-fill schema paths are separate systems covering overlapping ground** — copy_struct_from_user-shaped syscalls (clone3, sched_setattr, mount_attr, ...) use csfu.c's bucket model for the *outer* size word, while ARG_STRUCT_PTR_IN's schema-aware fill (struct_fill.c) covers arbitrary struct field content. A syscall using both needs its sanitise callback to coordinate the size-field fixup (csfu.c's `size_field_off`/`size_field_width`) against whatever the schema fill wrote for the same struct, since csfu.c does not consult struct_catalog.

## Summary

Per-dispatch flow: `generate_syscall_args()` picks sanitise-vs-minicorpus-replay, then `generic_sanitise()` walks each arg slot through `fill_arg()`, which dispatches via `argtype_table[]` to one of ~50 generators. Scalar/address/range/bitmask argtypes generate directly; struct-pointer argtypes (`ARG_STRUCT_PTR_IN/OUT/INOUT`) look up a `struct_catalog` descriptor and run the 3-pass schema-aware filler (struct_fill.c) followed by a 12%-gated single-field mutation (struct_mutate.c). Two defensive passes run after generation: `blanket_address_scrub`/`nested_address_scrub` (scrub.c) redirect any pointer aliasing shared memory or the libc heap, and `validate_arg_coupling` rejects a handful of provably-DOA cross-arg shapes before the kernel is entered. A cross-cutting CMP-hint injection mechanism (cmp_hint_inject.c) lets six-plus generators opportunistically substitute kernel-observed comparison constants for randomly-drawn values, rate-gated by an A/B baseline and plateau-driven amplification shared with the strategy/ and cmp_hints/ subsystems.
